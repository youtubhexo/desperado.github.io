项目需求分析

Posted on 2024-08-08

需求分析的重要性

需求分析是项目管理的重要组成部分，需要综合考虑时间、成本、质量三大要素。通过明确项目目标与范围，需求分析可以有效地指导项目的执行与管理。

需求分析的内容

  1. 业务需求

    • 识别目标群体：需求是直接面对用户的，还是针对企业内部的。

  2. 功能需求分析

    • 对功能进行详细划分：例如首页模块、课程模块、关于我们模块、上新模块等。

  3. 非功能需求分析

    • 例如UGC（用户生成内容）和AIGC（AI生成内容）的审核需求。

需求分析需要明确的内容

  1. 输入输出

    • 明确项目的输入和输出内容，确保每个环节都在预期范围内。

  2. 关联人及关联平台

    • 确定与项目相关的所有人员及平台，确保沟通顺畅。

  3. 最终效果与业务边界

    • 明确项目的最终目标与业务部分的边界，防止项目范围蔓延。

传统需求分析 vs 综合项目需求分析

  1. 传统需求分析

    • 以技术为主导，为了完成技术而开展工作，往往偏重技术实现而忽视业务需求。
    • 业务逻辑不清晰，知识点分散，综合应用能力不足。
    • 缺乏整体思考，对需求分析的工作流缺乏理解。

  2. 综合项目需求分析

    • 注重业务理解，明确需求，有助于提高开发效率。

需求分析中的常见问题

  1. 未对团队所有人明确需求

    • 需求未能清晰传达，导致团队成员对项目理解不一致。

  2. 忽视用户需求、需求变化与非功能性需求

    • 没有充分考虑用户需求的变化及非功能性需求，容易导致项目后期出现问题。

  3. 过度设计

    • 没有明确需求的优先级，导致项目资源浪费，影响交付效率。

前端技术的演进

让我们回到10年前，回顾前端世界的起点。最初，前端开发主要依赖于 HTML 和 CSS 绘制界面：

  • HTML: 基础标记语言，可以在1-2天内学会。
  • CSS: 用于样式设计，其学习和掌握程度取决于个人的审美和技巧。

AJAX 的登场

随着 AJAX 技术的出现，网页可以实现局部刷新，使得 JavaScript 逐渐变得不可或缺。

jQuery 的时代

jQuery 的出现简化了 DOM 操作，虽然现在的老旧项目中可能仍有使用，但它毕竟是一个时代的产物。

MVC 设计模式

随着前端需求的提升，我们需要简化代码并提高可维护性，于是 MVC 设计模型 应运而生，强调控制器、模型和视图层的分离。

AngularJS 作为这一时期的代表框架，因其组件化和双向绑定思想而备受瞩目，但由于其组件化和状态管理做得不够完善，最终被弃用。

三大框架的崛起

AngularJS 的失败为后来的三大框架提供了宝贵的借鉴，React、Vue 和 Angular2+ 采用了单向数据流和成熟的状态管理体系，成为前端技术的主流。

时代的见证者

前端技术是激进且不断演进的，前端开发者需要有危机意识，自律才能带来自由。不断革新自己，拥抱变化，是前端开发的永恒主题。

React设计哲学

简洁之美

React 的设计哲学可以总结为三个核心要素：

  • 单向数据流: 数据与页面绑定，单向渲染，表现为 UI = f(x)。
  • 虚拟DOM: 学习快照的思想，提高了性能。
  • 组件化: 保证代码的一致性，便于团队协作。

React 的优势

  • 可预测性: 编写可预测、符合开发者习惯的代码。
  • 简化模型: 使用 JSX、简化界面更新、坚持单向数据流。
  • 衍生项目: React Native，实现“一次学习，多处书写”。

Vue 的模板语法

Vue.js 使用基于 HTML 的模板语法，允许开发者声明式地将 DOM 绑定至底层 Vue 实例的数据。所有 Vue.js 的模板都是合法的 HTML，因此能被浏览器和 HTML 解析器解析。

在底层，Vue 将模板编译成虚拟 DOM 渲染函数。结合响应系统，Vue 能智能计算出最少的重新渲染组件数量，减少 DOM 操作次数。

对于熟悉虚拟 DOM 且偏爱 JavaScript 的开发者，可以直接编写渲染函数，使用可选的 JSX 语法。

学习模板语法

掌握 Vue.js 常见指令需要花费一些时间，但这对于编写高效的代码是必要的。

魔法上网、科学上网是指什么？

科学上网和魔法上网是指通过特定技术手段（如VPN）绕过网络限制访问国际互联网内容。由于存在网络防火墙（GFW），这种需求应运而生。

为什么有科学上网？

  • 因为存在“墙”（GFW）。

为什么有墙？

  • 因为“小孩子没有分辨能力”。

1. 了解是什么

  • 虚拟专用网络 (VPN) 是一种通过加密技术确保数据传输安全的技术。VPN在1998年左右与GFW的建立一起走入公众视野。

  • 个人使用VPN是否合法？

    • 身份是否合法：如外贸、学者、新闻媒体等合规使用者。
    • 使用目的是否纯净：如学术研究、与外企沟通等非敏感用途。
    • 是否满足政府监控的要求：内容是否符合监管要求。

2. 了解什么事情不能做

  • 不要传播：看到新奇事物时，不要在微信、QQ等媒体上转发，自己看即可。
  • 不参与政治讨论：阅读内容时，请不要参与政治相关评论或宣扬。
  • 遵守法律：购买和售卖VPN是违法行为，搭建并分享私人VPN也是违法的。
  • 禁止从事非法活动：包括但不限于HDD、下载分享盈利或诈骗等行为。

3. 机场运行原理探究

  • 前端：用户交互界面设计，如何确保用户体验和安全性。
  • 后端：服务器配置、流量管理、数据加密等技术分析。
  • 运营分析：机场服务的运营通常涉及从用户支付的订阅费用中抽取佣金。然而，第三方支付平台收取的手续费会直接影响到整体利润。因此，运营方需要权衡抽佣比例和支付成本，以维持服务的可持续性。

4. 成本预估

  • 以搬瓦工VPS为例，其配置如下：
    • CPU：2 核
    • 内存：2 GB
    • 硬盘：40 GB SSD
    • 流量：2000 GB/月
    • 带宽：高达 2.5 Gbps

5. 法律知识及行政复议

法律知识

  • 《计算机信息网络国际联网管理暂行规定》
  • 《国际通信出入口局管理办法》第十九条
  • 《中华人民共和国网络安全法》：第四十四条、六十三条
  • 《中华人民共和国刑法》：第二百四十七条

从过去的时光里，窥探layui

Posted on 2024-07-30

layui 只是绕过了 CSS，方便后端程序员能快速完成管理界面开发。作为一个前端开发者，一个是 CSS 要熟练掌握，一个是 JS 框架的掌握。

前端的发展是一个激进的过程，所以要求前端工程师必须革新与拥抱变化。

例如：从 Vue 发展来看，从 Vue2 到 Vue3 的发展，从 Webpack 到 Vite 打包工具的出现，从 vue-cli 到 Vite 脚手架的变化。

短短几年的时间，发展就这么快，不像后端一样，一潭死水。很有意思，后端借助一些模板项目，比如 layui 打算取代前端，而前端又可以借助 NestJS 来实现后端架构，世界就是一个矛盾又辩证的关系。

那么说到前端，我们可以从项目搭建开始说起。

借助一个脚手架或内部模板项目，进行项目初始化。在此过程中，需要关注主流的技术，是否符合项目的需求，技术栈是否成熟。我们假定已经理清了项目需求，那么从技术的角度来看，阅读文档的能力非常重要。

寻找你的核心竞争力

  1. 是否遇到的技术困难找到了一种解决方法后，还会继续思考新方法，这个是当然的，我们就是靠脑子吃饭的。
  2. 是否保持学习新的语言新的技术，并尝试在适当的场景下应用，这是生死存亡的大事。
  3. 对于团队中不一样的声音保持着警醒与包容（这是我所缺少的）。

以 AI 技术浪潮为例，我不会从头到尾地去研究 AI 技术，除非热爱。我会看看大 V 怎么，有什么已有的创意产品，我能做什么。

关于新技术的学习思路

  1. 关注技术相关的趋势与热点，通过博客论坛了解最新动态（RSS 订阅）。
  2. 学习官方文档及源码，了解实现原理与机制。
  3. 参与技术交流活动（社交媒体、线下活动等）、参与项目分享实践。

希望大家多多支持我，继续看我以后书写博客的能力。
